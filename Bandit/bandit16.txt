1. Goal

The credentials for the next level can be retrieved by submitting the password of the current level to a port on localhost in the range 31000 to 32000. First find out which of these ports have a server listening on them. Then find out which of those speak SSL and which donâ€™t. There is only 1 server that will give the next credentials, the others will simply send back to you whatever you send to it.

2. Commands

ssh, telnet, nc, openssl, s_client, nmap

3. Solution

First, like previously lets get current password

bandit16@bandit:/tmp/kutas$ cat /etc/bandit_pass/bandit16 
cluFn7wTiGryunymYOu4RcffSxQluehd

Then I need to find one port from range 31000-32000, let's use nmap with -sT (TCP connect, no sudo needed contrary to -sS)

nmap -sT -p 31000-32000 localhost

Starting Nmap 7.40 ( https://nmap.org ) at 2020-04-15 22:39 CEST

Host is up (0.00024s latency).

PORT      STATE    SERVICE
31518/tcp filtered unknown
31790/tcp open     unknown

Nmap done: 1 IP address (1 host up) scanned in 1.25 seconds

 I see that 31518 is filtered and 31790 not. I need to speak with SSL, so let's try same command as in previous exercise:

 echo cluFn7wTiGryunymYOu4RcffSxQluehd | openssl s_client -quiet -connect localhost:31790
depth=0 CN = localhost

verify return:1
depth=0 CN = localhost
verify return:1
Correct!

It gives me private key. I have to send it to file but cannot do it in this directory but /tmp, so

mkdir /tmp/myname && cd /tmp/myname
touch ssh.private
vim sshkey.private

Then I copy showed key to file, :wq! to save and quit. Next step is to give correct rights so:

chmod 600 ssh.private

and finally I use same  command as in one of previous, so:

ssh -i ssh.private bandit17@localhost
